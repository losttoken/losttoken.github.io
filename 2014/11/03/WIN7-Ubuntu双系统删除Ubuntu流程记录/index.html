WIN7-Ubuntu双系统删除Ubuntu流程记录

之前是Ubuntu12.04和Windows7双系统，安装时为了性能并没有选择wubi安装方式，重新分了ext系列分区，导致windows的启动依赖于linux的grub，现需要干掉ubuntu，节省出磁盘空间。干掉ubuntu最重要的就是能正确引导windows：

  1. 进入win7，下载个软件MbrFix，放在C:\windows\system32文件夹中

  2. 点击开始》所有程序》附件》命令提示符

  3. 在命令提示符中输入 

    MbrFix /drive 0 fixmbr /yes

4. 此时已经修复了MBR，重启电脑后会发现已经没有了linux启动选项，直接进入windows了

  1. 在 Win7下，在计算机(Computer)上点右键->管理(Manage)->磁盘管理(DiskManagement)：有2个没有卷标名称(卷标名称就是驱动器号，如C:,D:,E:,…)的主分区(PrimaryPartition)就是Ubuntu的分区，删除卷，包括swap之类的全部干掉，空闲磁盘就合并成一个

  2. 这时候新建卷有可能会提示“磁盘上没有足够的空间完成操作”，是因为不是动态磁盘不能执行分卷之类的磁盘操作，这时，需要使用“Windows+R”调出运行对话框。输入“Diskpart”，调出命令行，输入“list disk” ，找出没分配的那个disk，比如是0，输入Select Disk 0，选择该disk后输入convert dynamic，将磁盘转换成动态磁盘

  3. 这是重新回到磁盘管理，刚才那个空闲的绿色色块代表的分区变成了棕黄色和黑色，这是执行新加卷、格式化之类的就不会有任何阻碍了

    新分区后可以了，干掉了Ubuntu，拿出了90G的磁盘空间

    补充：

以上的方法重启后无法引导系统，感觉应该是那步动态磁盘的转换导致的，磁盘转换成了动态磁盘，同时，之前的MBR引导信息找不到新的动态磁盘了，下载PE系统到U盘，进入PE系统修复下MBR引导就可以了。

重新进入系统后，多了个 earlier version of windows ，在命令行，执行： 

bcdedit /delete {ntldr} /f

上面那个教程正确的做法应该是用动态磁盘分好区后再执行一遍 

MbrFix /drive 0 fixmbr /yes

，就不会出现后面的故障了
